浅述Java

发表于 2017-02-17 | 分类于 Java |

使用java这些年来，印象最深的一个词就是：对象（多么的深意）。是的，它就是一门面向对象的编程语言，具有简单性、面向对象、分布式、健壮性、安全性、平台独立与可移植性、多线程、动态性等特点。

开发环境、工具

JDK环境

JDK是Java软件开发的工具包，是Java开发的核心，包括了运行环境及核心类库，其版本也从最开始的1.1到现在的1.8。其配置环境如Windows，需设置\autoexec.bat文件：
set JAVA_HOME=JDK安装目录
set PATH=%JAVA_HOME%\bin;%PATH%
set CLASSPATH=.;%JAVA_HOME%\lib
也可右键修改系统环境变量，详情操作可自行Google

Linux下修改.bashrc文件（此法更为安全）：
export JAVA_HOME=/home/app/jdk1.7.0_60
export CLASSPATH=.:$JAVA_HOME/lib:$JAVA_HOME/jre/lib:$CLASSPATH
export PATH=$JAVA_HOME/bin:$JAVA_HOME/jre/bin:$PATH

JDK常用组件

javac：编译器，将源程序转成字节码
jar：打包工具，将相关的类文件打包成一个文件
javadoc：文档生成器，从源码注释中提取文档
jdb：debugger，查错工具
java: 运行编译后的java程序（.class后缀的）
appletviewer：小程序浏览器，一种执行HTML文件上的Java小程序的Java浏览器

常用包

java.lang： 系统基础类，比如String
java.io： 输入输出类，比如文件操作
java.nio：为了完善io包中的功能，提高io包中性能而写的一个新包 ，例如NIO非堵塞应用
java.net: 网络有关的类，比如URL，URLConnection等
java.util： 系统辅助类，特别是集合类Collection，List，Map等
java.sql： 数据库操作的类，Connection， Statement，ResultSet等
javax.servlet：JSP，Servlet等类

工具

可选工具有：Eclipse，myeclipse，IntelliJ IDEA，选择性都因人而异。

工作原理

Java源程序经过编译器编译后变成字节码，字节码由虚拟机解释执行，虚拟机将每一条要执行的字节码送给解释器，解释器将其翻译成特定机器上的机器码，然后在特定的机器上运行。
简易原理图：
.java→.class→机器码
java编译器 (编译) → 虚拟机(解释执行) → 解释器(翻译) → 机器码

数据类型：

数据型-byte(8),short(16),int(32),long(64)
浮点型-float(32),double(64)
字符型-char(16)
布尔型-boolean

框架 技术

Java常见框架包括：Hibernate、Struts2、spring、iBATIS等，技术分类J2EE、J2SE、J2ME。

Hibernate

Hibernate是一个开放源代码的ORM(对象关系映射)框架，对JDBC进行了非常轻量级的对象封装，使开发人员可以用面向对象编程思维来操纵数据库。 Hibernate的API一共有6个，分别为:Session、SessionFactory、Transaction、Query、Criteria和Configuration。通过这些接口，可以对持久化对象进行存取、事务控制。可以应用在任何使用JDBC的场合。

Struts2

Struts2是一个基于MVC设计模式的Web应用框架，主要采用Servlet和JSP技术来实现，把Servlet、JSP、自定义标签和信息资源(message resources)整合到一个统一的框架中，极大的节省了时间。

Spring

Spring的架构是基于JavaBean属性的Inversion of Control容器，也包括了JDBC框架，在数据访问架构还集成了Hibernate和其他O/R mapping解决方案。同时提供事务管理抽象，为不同的事务API提供一致的编程模型，如JTA、JDBC、Hibernate、iBATIS数据库层 。Spring还提供了可以和IoC容器集成的强大而灵活的MVC Web框架。

iBATIS

iBATIS是一个‘半自动’orm框架，相对Hibernate等 ‘全自动’ORM机制而言，ibatis 以SQL开发的工作量和数据库移植性上的让步，则为系统 设计提供了更大的自由空间。

分类

Java包括了J2EE、J2SE、J2ME三个分支。
Java EE（Java Platform，Enterprise Edition）：j2ee是在j2se的基础上构建的，它提供 Web 服务、组件模型、管理和通信 API，可以用来实现企业级的面向服务体系结构（service-oriented architecture，SOA）和 Web 2.0 应用程序。一般为服务器端程序应用。
Java SE（Java Platform，Standard Edition）：开发和部署在桌面、服务器、嵌入式环境和实时环境中使用的 Java 应用程序。j2se包含了支持 Java Web 服务开发的类，并为j2ee提供基础。一般应用于客户端。
Java ME（Java Platform，Micro Edition）：包括灵活的用户界面、健壮的安全模型、许多内置的网络协议以及对可以动态下载的连网和离线应用程序的丰富支持，一般位于嵌入式应用。

其中，j2ee包括了：
1、jdbc：数据库连接方式
2、jsp：动态网页
3、ejb：提供开发和实施分布式商务逻辑
4、jms：中间件相互通信的应用程序接口
5、servlet：提供的功能大多与JSP类似，只是实现的方式不同
6、xml：用来定义其它标记语言的语言
7、jndi：用于名字和目录服务，提供了一致的模型来存取和操作企业级的资源如DNS和LDAP，本地文件系统等
8、jts：规定了事务管理器的实现方式
9、jta：存取各种事务
